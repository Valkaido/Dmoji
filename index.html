Démarrage rapide

Une installation fonctionnelle demande simplement le bot sur votre serveur et un message de test valide. Commencez ici avant de consulter la référence complète.

1. Inviter Dmoji sur le serveur

Ajoutez le bot au serveur où vous voulez utiliser les émotes et terminez l'invitation avec les autorisations nécessaires pour qu'il puisse renvoyer correctement les messages.

2. Vérifier les émotes disponibles

Lancez /emotes dans le salon cible. Le sélecteur affiche la liste actuelle des émotes, la recherche, vos derniers choix et les favoris déjà enregistrés sur votre compte.

3. Envoyer un message de test

Publiez un message Discord normal contenant un ou plusieurs codes d'émote. Si tous les codes sont valides, Dmoji renvoie immédiatement le message avec votre profil visible dans ce salon.

Commandes

L'ensemble des commandes reste volontairement réduit. Au quotidien, tout se fait surtout directement dans les messages, tandis que les commandes /emotes et /info servent à chercher, envoyer et vérifier plus facilement.

/emotes

Le sélecteur est le moyen le plus rapide pour parcourir la liste, revoir vos favoris et envoyer une émote sans mémoriser son nom exact.

/emotes nom:<émote> taille:<taille> texte:<message>

Utilisez cette version avec options lorsque vous voulez choisir une émote précise, une taille et un texte optionnel dans une seule action.

/info

Cet outil aide les modérateurs et les utilisateurs à retrouver quelle émote a été utilisée dans un message précédemment renvoyé par Dmoji.

Fonctionnement

Dmoji est conçu pour rester discret. L'utilisateur écrit un seul message, le bot le vérifie, puis le salon reçoit une nouvelle version uniquement si tout peut être traité correctement.

  1. Dmoji cherche les codes d'émote dans le message. Ces codes commencent par / et peuvent inclure une taille, par exemple :2x ou :4x.
  2. Chaque code doit être reconnu avant l'envoi. Si un seul nom d'émote est inconnu ou mal écrit, le bot ne renvoie rien afin d'éviter un résultat incomplet.
  3. Le message d'origine est supprimé puis remplacé par une nouvelle version. Le message renvoyé garde le texte autour des émotes, reprend le nom affiché de l'auteur et s'adapte à ce que Discord peut afficher à cet endroit.
  4. Les pièces jointes sont conservées si elles peuvent encore être récupérées et renvoyées. Les fichiers trop volumineux, les liens expirés ou un accès restreint aux pièces jointes peuvent bloquer cette étape.

Remarque Dmoji ne publie pas d'erreur visible quand une tentative échoue. Cela garde les salons lisibles et évite de transformer une simple faute de frappe en spam du bot.

Dépannage

La plupart des échecs viennent de trois causes : un code d'émote inconnu, une autorisation manquante ou une pièce jointe que Discord ne laisse pas renvoyer. Vérifiez les cas ci-dessous avant d'aller plus loin.

Le message ne change pas après l'envoi.

Problème Dmoji ne touche jamais au message.

Cause probable Le message ne contient pas de code d'émote valide ou le bot ne peut pas lire le salon.

Correction Vérifiez que le code commence bien par /, correspond exactement à une émote existante et que le bot peut voir le salon.

Le message d'origine disparaît, mais rien n'est republié.

Problème L'étape de réécriture échoue après la suppression.

Cause probable Il manque au bot les permissions Envoyer des messages, gérer l'envoi au bon nom ou l'accès nécessaire aux pièces jointes.

Correction Revérifiez les permissions du salon, puis refaites un test avec un simple message texte plus émote, sans fichier joint.

Une seule émote invalide bloque tout le message.

Problème Rien n'est republié alors qu'une ou plusieurs émotes semblent correctes.

Cause probable Dmoji exige que chaque code du message soit reconnu avant de renvoyer la nouvelle version.

Correction Supprimez le code inconnu, testez chaque nom d'émote séparément puis renvoyez le message une fois que tout est valide.

Les pièces jointes manquent dans le message republié.

Problème Le texte et les émotes sont republiés, mais pas les fichiers.

Cause probable La pièce jointe source est trop volumineuse, expirée ou inaccessible au moment du renvoi.

Correction Recommencez avec un fichier plus léger, laissez les permissions d'accès aux fichiers ouvertes et vérifiez que le bot peut téléverser des pièces jointes.

Le bot fonctionne dans les salons, mais pas dans les fils ou les forums.

Problème Le bot échoue uniquement dans les fils ou les forums.

Cause probable Le salon principal n'accorde pas au rôle du bot les permissions nécessaires dans les fils ou pour renvoyer correctement les messages.

Correction Vérifiez le salon parent, pas seulement le fil, et assurez-vous que l'option Envoyer des messages dans les fils est activée.

FAQ

Voici les questions qui reviennent le plus souvent après le premier test concluant.

Dmoji publie-t-il des erreurs visibles lorsqu'un code est invalide ?

Non. Les réécritures échouées restent silencieuses afin qu'une émote mal saisie n'ajoute pas de bruit inutile dans le salon.

Puis-je garder du texte normal autour d'une émote ?

Oui. Dmoji conserve le texte avant, entre et après les codes valides tant que le message complet est reconnu correctement.

Les émotes 7TV peuvent-elles se comporter exactement comme les émojis natifs de Discord ?

Non. Dmoji réécrit le message pour simuler le résultat, mais les émotes externes ne sont pas insérées comme de vrais émojis natifs dans le message d'origine.

Les favoris et les émotes récentes sont-ils partagés avec tout le serveur ?

Non. Le sélecteur est pensé pour accélérer l'usage de chaque personne, donc les favoris et les éléments récents restent personnels.

Les noms spéciaux comme /D: fonctionnent-ils ?

Oui, à condition que la liste d'émotes configurée contienne exactement ce nom et que le code écrit corresponde parfaitement.

Dmoji est un bot Discord dédié aux émotes : il réécrit les messages avec les émotes configurées, le texte conservé et le transfert optionnel des pièces jointes.
